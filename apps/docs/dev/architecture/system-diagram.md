## Data Flow: Note Edit

1. User types in editor вҶ’ TipTap generates ProseMirror transaction
2. Yjs encodes the change as a CRDT update
3. Update is broadcast over WebSocket to all connected clients
4. Server receives update вҶ’ broadcasts to all other clients in the room
5. Server debounces (500ms) вҶ’ persists Markdown to filesystem
6. Server updates note metadata in PostgreSQL (title, modified_at, search index)

## Request Flow: REST API

```
Browser вҶ’ Nginx вҶ’ Next.js (API Route proxy) вҶ’ NestJS API
       OR
Browser вҶ’ Nginx вҶ’ NestJS API (direct /api/* calls)
```
